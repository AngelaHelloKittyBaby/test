## ✨ 功能特点

- 🆓 **免费API**：使用和风天气免费接口，无需备案
- ⏰ **定时任务**：支持自定义时间间隔自动获取
- 🌍 **中文城市**：支持中文城市名（北京、上海、广州等）
- 📊 **信息完整**：温度、天气、风向、湿度、体感温度等
- 🛡️ **异常处理**：网络错误、API错误等全面捕获
- 📝 **日志记录**：可选保存历史天气到本地文件
- 💻 **跨平台**：兼容 Windows/macOS/Linux

## 📦 安装步骤

### 1. 安装 Python
确保已安装 Python 3.7 或更高版本：
```bash
python --version
```

### 2. 安装依赖库
在项目目录下运行：
```bash
pip install -r requirements.txt
```

或手动安装：
```bash
pip install requests schedule
```

### 3. 配置文件
编辑 `config.py` 文件，填写您的 API 密钥和目标城市。

## 🔑 API 密钥申请教程

### 步骤 1：注册账号
1. 访问 [和风天气开发平台](https://dev.qweather.com/)
2. 点击右上角"注册"，填写邮箱和密码
3. 登录邮箱验证账号

### 步骤 2：创建应用
1. 登录后点击顶部"控制台"
2. 左侧菜单选择"应用管理"
3. 点击"创建应用"按钮
4. 填写应用名称（如"我的天气小程序"）
5. 选择"免费订阅"（Free Subscription）

### 步骤 3：获取密钥
1. 应用创建成功后，点击应用名称进入详情
2. 找到 **Public ID** 旁边的 **Key**
3. 复制这个密钥（一串字母数字组合）

### 步骤 4：填入配置
将复制的密钥填入 `config.py`：
```python
API_KEY = "您的实际API密钥"
```

> - 每日请求次数：1000 次/天
> - 每分钟请求次数：100 次/分钟
> - 对于个人使用完全足够！

## 🚀 使用说明

### 修改配置

编辑 `config.py` 文件：

```python
# 必填：API密钥（从和风天气获取）
API_KEY = "your_api_key_here"

# 必填：目标城市（支持中文）
CITY_NAME = "北京"

# 可选：定时间隔（分钟）
INTERVAL_MINUTES = 30

# 可选：是否保存日志
SAVE_LOG = True
```

### 运行程序

```bash
python weather_app.py
```

### 停止程序

按 `Ctrl + C` 即可停止程序运行。

## 📖 代码功能解释

### 文件结构

```
weather_project/
├── weather_app.py      # 主程序文件
├── config.py           # 配置文件
├── requirements.txt    # 依赖库列表
├── weather_log.txt     # 天气日志（自动生成）
└── README_WEATHER.md   # 使用说明
```

### 核心函数说明

#### 1. `get_weather_data(city, api_key)`
- **功能**：调用和风天气API获取天气数据
- **参数**：
  - `city`: 城市名称（中文）
  - `api_key`: API密钥
- **返回**：包含天气信息的字典，失败返回None
- **异常处理**：网络超时、连接错误、JSON解析错误等

#### 2. `format_weather_output(weather_data)`
- **功能**：将天气数据格式化为美观的字符串
- **参数**：`weather_data` - 天气数据字典
- **返回**：格式化后的多行字符串

#### 3. `save_weather_log(weather_data, log_file)`
- **功能**：将天气信息保存到本地日志文件
- **参数**：
  - `weather_data`: 天气数据字典
  - `log_file`: 日志文件路径（默认 weather_log.txt）

#### 4. `fetch_and_display_weather()`
- **功能**：获取并显示天气的主函数
- **说明**：这是定时任务调用的核心函数

#### 5. `setup_schedule()`
- **功能**：设置定时任务
- **说明**：根据 `INTERVAL_MINUTES` 配置创建定时任务

#### 6. `main()`
- **功能**：程序主入口
- **流程**：
  1. 检查配置有效性
  2. 打印程序信息
  3. 设置定时任务
  4. 进入主循环等待任务执行

## 🔧 常见问题解决

### 问题 1：API 调用失败，错误码 401

**原因**：API 密钥无效或未激活

**解决**：
1. 检查 `config.py` 中的 `API_KEY` 是否填写正确
2. 确认和风天气应用已创建并启用
3. 新创建的密钥可能需要几分钟才能生效，等待后重试

### 问题 2：城市查找失败，错误码 404

**原因**：城市名称不正确

**解决**：
1. 使用标准中文城市名（如"北京"而非"北京市"）
2. 检查是否有错别字
3. 尝试使用省会城市名称

### 问题 3：程序运行但没有定时输出

**原因**：`schedule` 库需要持续运行

**解决**：
1. 确保程序处于运行状态，不要关闭终端
2. 检查 `INTERVAL_MINUTES` 设置是否合理（不要太长）
3. 程序启动后会立即执行一次，之后按间隔执行

### 问题 4：中文显示乱码

**原因**：终端编码问题

**解决**：
- **Windows**：使用 PowerShell 或 CMD 运行，确保使用 UTF-8 编码
- **macOS/Linux**：终端默认支持 UTF-8，通常无问题
- 在代码开头已添加 `# -*- coding: utf-8 -*-` 声明

### 问题 5：网络请求超时

**原因**：网络连接问题或API服务异常

**解决**：
1. 检查网络连接是否正常
2. 确认能访问 https://dev.qweather.com/
3. 可能是临时网络问题，程序会自动重试

### 问题 6：ModuleNotFoundError: No module named 'requests'

**原因**：依赖库未安装

**解决**：
```bash
pip install requests schedule
```
