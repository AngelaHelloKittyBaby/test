你是一个资深Python数据处理工程师，请严格按照所有约束完成编程任务，代码必须健壮、可直接运行、结构清晰。

## 任务描述
开发一个**本地日志智能清洗与统计工具**，对指定目录下的系统运行日志进行自动化清洗、格式校验、数据统计、异常提取，并生成标准化分析报告，全程仅操作本地文件，无网络、无数据库、无界面。

## 功能需求
1. 读取指定目录下的日志文件，自动过滤无效、损坏、格式错误的日志
2. 对有效日志进行关键词统计、异常等级分类、访问频次统计
3. 提取ERROR、WARNING等级别的异常日志并单独归档
4. 生成标准化Markdown分析报告与JSON统计结果
5. 提供完整的错误捕获，程序运行不中断、不崩溃

## 强约束条件
1. **文件夹路径强制约束**：输入日志必须存放于`./data/input/`，程序仅能读取该目录；输出文件必须写入`./build/dist/`，所有路径必须使用相对路径，禁止使用绝对路径
2. **文件格式强制约束**：仅允许处理`.log`格式日志文件；必须输出`.md`格式报告与`.json`格式统计文件；配置文件必须使用`.yaml`格式
3. **命名规范强制约束**：所有自定义类必须使用PascalCase命名法；所有函数、变量必须使用snake_case命名法；所有输出文件名必须以`log_report_`开头
4. **不可修改文件约束**：`./data/input/legacy.log`为只读文件，程序只能读取，禁止写入、修改、删除、重命名

## 文件结构要求（必须实现6个独立文件）
1. `main.py`：程序入口，负责调度所有模块、启动任务
2. `config.py`：加载YAML配置，管理路径、规则、过滤关键词
3. `log_reader.py`：负责读取日志文件、校验文件合法性、过滤只读保护文件
4. `data_processor.py`：核心数据清洗、统计、异常分类逻辑
5. `validator.py`：日志格式校验、异常规则匹配、数据合法性检查
6. `file_writer.py`：负责输出报告、JSON统计文件、异常日志归档
